# Pymarkdown Configuration

This repository uses `.pymarkdown` (JSON format) to configure markdown linting rules.

## Current Configuration

### MD013 (Line Length)

- **Status**: Enabled with limit of 210 characters
- **Reason**: Accommodates tables with long badge URLs while still catching overly long prose
- **Default**: 80 characters

### MD036 (Emphasis as Heading)

- **Status**: Disabled
- **Reason**: Causes false positives with bold text in numbered lists (e.g., `1. **Step title**`)
- **Default**: Enabled

## Per-Line Exceptions

Unfortunately, pymarkdown does not currently support inline HTML comments like `<!-- markdownlint-disable -->`.

If you need per-line exceptions, you have three options:

1. **Adjust the global rule** in `.pymarkdown` (current approach)
2. **Use command-line args** in `.pre-commit-config.yaml` to disable rules entirely
3. **Split content** across multiple files with different rules

## Configuration Format

The `.pymarkdown` file uses JSON:

```json
{
  "plugins": {
    "rule-id": {
      "enabled": true/false,
      "parameter": value
    }
  }
}
```

See [pymarkdown documentation](https://github.com/jackdewinter/pymarkdown) for available rules and parameters.
